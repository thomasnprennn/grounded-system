# C1 — Scope Mapping

## Purpose

Define exactly what implementation includes before pricing or timeline is locked.

## Inputs Needed

- approved scope
- approved brand system
- client assets
- platform access
- timeline
- dependencies

## Files to Read

- AGENTS.md
- policies/scope-boundaries.md
- policies/approval-policy.md
- workflows/B-brand-system-build/00-overview.md

## Output to Create

scope map, exclusions, dependencies, timeline risk

## Save To

`/clients/[client]/implementation/01-scope-mapping.md`

## Human Review

Human validates scope, pricing, deliverable feasibility, and client communication.

## Quality Gate

- scope clear
- approval checkpoint clear
- no hidden deliverables
- dependencies marked
- final handoff usable

## AI Instruction

```txt
Map the implementation scope. Separate must-have, optional, excluded, dependency, and risk. Do not quote without clear scope.
```
