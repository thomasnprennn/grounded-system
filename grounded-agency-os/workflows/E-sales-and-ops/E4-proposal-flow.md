# E4 — Proposal Flow

## Purpose

Foundation Audit can be closed with clear DM + scope. Brand System Build and Full Implementation need proposal document with scope, timeline, price, payment terms, revision policy, and next step.

## Output

Save relevant messages, decisions, and next steps inside:

`/clients/[client]/messages/`
`/clients/[client]/approvals/`
`/clients/[client]/03-decisions.md`

## Quality Gate

- one clear next step
- no pressure language
- no vague pricing
- package matched to problem
- client decision documented
